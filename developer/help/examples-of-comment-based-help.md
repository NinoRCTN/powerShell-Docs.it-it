# <a name="examples-of-comment-based-help"></a>Esempi della Guida basata sui commenti

In questo argomento include l'esempio che illustrano come usare la Guida basata su commenti per funzioni e script.

## <a name="example-1-comment-based-help-for-a-function"></a>Esempio 1: Guida basata su commenti per una funzione

 La funzione di esempio seguente include la Guida basata sui commenti.

```powershell
function Add-Extension
{
    param ([string]$Name,[string]$Extension = "txt")
    $name = $name + "." + $extension
    $name

    <#
        .SYNOPSIS
        Adds a file name extension to a supplied name.

        .DESCRIPTION
        Adds a file name extension to a supplied name.
        Takes any strings for the file name or extension.

        .PARAMETER Name
        Specifies the file name.

        .PARAMETER Extension
        Specifies the extension. "Txt" is the default.

        .INPUTS
        None. You cannot pipe objects to Add-Extension.

        .OUTPUTS
        System.String. Add-Extension returns a string with the extension or file name.

        .EXAMPLE
        C:\PS> extension -name "File"
        File.txt

        .EXAMPLE
        C:\PS> extension -name "File" -extension "doc"
        File.doc

        .EXAMPLE
        C:\PS> extension "File" "doc"
        File.doc

        .LINK
        Online version: http://www.fabrikam.com/extension.html

        .LINK
        Set-Item
    #>
}
```

L'output seguente mostra i risultati di un comando Get-Help che consente di visualizzare la Guida per la funzione Aggiungi estensione.

```powershell
C:\PS> get-help add-extension -full
```

## <a name="example-2-comment-based-help-for-a-script"></a>Esempio 2: Guida basata su commenti per uno Script

La funzione di esempio seguente include la Guida basata sui commenti.

Si noti che le righe vuote tra la chiusura **#>** e il `Param` istruzione. In uno script che non ha un `Param` istruzione, deve esserci almeno due righe vuote tra il commento finale dell'argomento della Guida e la prima dichiarazione di funzione. Senza queste righe vuote, Get-Help associa l'argomento della Guida con la funzione, anziché lo script.

```powershell
<#
  .SYNOPSIS
  Performs monthly data updates.

  .DESCRIPTION
  The Update-Month.ps1 script updates the registry with new data generated
  during the past month and generates a report.

  .PARAMETER InputPath
  Specifies the path to the CSV-based input file.

  .PARAMETER OutputPath
  Specifies the name and path for the CSV-based output file. By default,
  MonthlyUpdates.ps1 generates a name from the date and time it runs, and
  saves the output in the local directory.

  .INPUTS
  None. You cannot pipe objects to Update-Month.ps1.

  .OUTPUTS
  None. Update-Month.ps1 does not generate any output.

  .EXAMPLE
  C:\PS> .\Update-Month.ps1

  .EXAMPLE
  C:\PS> .\Update-Month.ps1 -inputpath C:\Data\January.csv

  .EXAMPLE
  C:\PS> .\Update-Month.ps1 -inputpath C:\Data\January.csv -outputPath C:\Reports\2009\January.csv
#>

param ([string]$InputPath, [string]$OutPutPath)

function Get-Data { }
```

Il comando seguente ottiene lo script Help. Poiché lo script non è n una directory in cui è elencata nella variabile di ambiente Path, il comando Get-Help che ottiene lo script Help deve specificare il percorso dello script.

```powershell
C:\PS> get-help c:\ps-test\update-month.ps1 -full
```

## <a name="example-3-parameter-descriptions-in-a-param-statement"></a>Esempio 3: Descrizioni dei parametri in un'istruzione Param

In questo esempio mostra come inserire parameterdescriptions nel `Param` istruzione di una funzione o script. Questo formato è particolarmente utile quando le descrizioni dei parametri sono brevi.

```powershell
function Add-Extension
{
    param
    (
        [string]
        # Specifies the file name.
        $name,

        [string]
        # Specifies the file name extension. "Txt" is the default.
        $extension = "txt"
    )
    $name = $name + "." + $extension
    $name

    <#
        .SYNOPSIS
        Adds a file name extension to a supplied name.
...
    #>
```

I risultati sono gli stessi dei risultati, ad esempio 1. Get-Help interpreta le descrizioni dei parametri come se si sono accompagnati dal `.Parameter` (parola chiave).

## <a name="example-4--redirecting-to-an-xml-file"></a>Esempio 4:  Il reindirizzamento a un File XML

È possibile scrivere argomenti della Guida basati su XML per le funzioni e script. Anche se la Guida basata sui commenti è più facile da implementare, Guida basata su XML è necessaria se si desidera maggiore controllo sul contenuto della Guida o se tradurre gli argomenti della Guida in più lingue. Nell'esempio seguente mostra le prime righe dello script di aggiornamento Month.ps1. Lo script Usa i `.ExternalHelp` (parola chiave) per specificare il percorso a un argomento basato su XML per lo script.

```powershell
#  .ExternalHelp C:\MyScripts\Update-Month-Help.xml

    param ([string]$InputPath, [string]$OutPutPath)

    function Get-Data { }
```

L'esempio seguente illustra l'uso del `.ExternalHelp` parola chiave in una funzione.

```powershell
function Add-Extension
{
    param ([string] $name, [string]$extension = "txt")
    $name = $name + "." + $extension
    $name

    # .ExternalHelp C:\ps-test\Add-Extension.xml
}
```

## <a name="example-5--redirecting-to-a-different-help-topic"></a>Esempio 5:  Il reindirizzamento a un altro argomento della Guida

Il codice seguente è tratto dall'inizio del predefiniti `Help` (funzione) in Windows PowerShell, che visualizza una schermata del testo della Guida alla volta. Poiché l'argomento della Guida per il cmdlet Get-Help viene descritta la funzione di Guida in linea, la funzione Help Usa la `.ForwardHelpTargetName` e `.ForwardHelpCategory` parole chiave per reindirizzare l'utente all'argomento della Guida del cmdlet Get-Help.

```powershell
function help
{

    <#
      .FORWARDHELPTARGETNAME Get-Help
      .FORWARDHELPCATEGORY Cmdlet
    #>
    [CmdletBinding(DefaultParameterSetName='AllUsersView')]
    param(
            [Parameter(Position=0, ValueFromPipelineByPropertyName=$true)]
            [System.String]
            ${Name},
    ...
```

Il comando seguente usa questa funzionalità. Quando un utente digita un comando Get-Help per la funzione Help, Get-Help Visualizza l'argomento della Guida per il cmdlet Get-Help.

```powershell
C:\PS> get-help help
```

```output
            NAME
                Get-Help

            SYNOPSIS
                Displays information about Windows PowerShell cmdlets and concepts.
            ...
```
